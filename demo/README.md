# Astronomical Object Classifier — Demo

Interactive Streamlit interface for the trained ResNet50 classifier. Upload
an astronomical image and see the predicted class plus a Grad-CAM heatmap
showing what the model looked at.

## Files

| File | Purpose |
|---|---|
| `app.py` | Streamlit UI (the entry point) |
| `model.py` | ResNet50 factory + checkpoint loader |
| `inference.py` | Preprocessing, end-to-end `predict()`, heatmap overlay |
| `gradcam.py` | Grad-CAM as a context manager |
| `requirements.txt` | Python dependencies |

## Run locally

```bash
pip install -r requirements.txt

# Point at your trained checkpoint (or place it at the default location).
export CHECKPOINT_PATH=path/to/baseline_resnet50.pt

streamlit run app.py
```

Open the URL it prints (usually `http://localhost:8501`).

If you don't have a checkpoint yet, train one with the notebook
(`astro_classifier_training.ipynb`) — even the synthetic-data run from the smoke test
produces a checkpoint the app can load.

## Default checkpoint path

`checkpoints/baseline_resnet50.pt` (relative to `app.py`). Override with the
`CHECKPOINT_PATH` environment variable.

## Deployment

### Streamlit Community Cloud (free)

1. Push this folder to a public GitHub repo.
2. At [share.streamlit.io](https://share.streamlit.io), connect the repo and
   point at `app.py` as the entry point.
3. The checkpoint file (~95 MB) is too large for git. Two options:
   - **Option A — Git LFS.** Track the .pt file with Git LFS, push, done.
     Simple but uses your LFS bandwidth quota.
   - **Option B — Download on startup.** Host the checkpoint on Hugging Face
     Hub or Google Drive, and add this snippet to the top of `app.py`:
     ```python
     import urllib.request
     CKPT_URL = "https://huggingface.co/<user>/<repo>/resolve/main/baseline_resnet50.pt"
     if not Path(CHECKPOINT_PATH).is_file():
         Path(CHECKPOINT_PATH).parent.mkdir(parents=True, exist_ok=True)
         urllib.request.urlretrieve(CKPT_URL, CHECKPOINT_PATH)
     ```
     Cleaner, but adds ~30 seconds to the first cold start.

### Local-only (laptop demo)

Just run `streamlit run app.py`. No deployment needed. Sufficient for the
project's "hosted demo or local inference interface" deliverable.

## What the app shows

- **Input image** — the uploaded file, displayed at full size.
- **Grad-CAM overlay** — the input with a jet-colormap heatmap superimposed.
  Bright = strong influence on the predicted class.
- **Predicted class** — top-1 prediction with a confidence bar.
- **All class probabilities** — full softmax distribution as a ranked list.
- **Low/high-confidence hints** — contextual notes when confidence is very low
  (< 50%) or very high (> 95%) to set user expectations.

## Notes for graders / evaluators

The Grad-CAM heatmap is the same one generated by section 11 of the training
notebook — same `GradCAM` class, same target layer (`model.layer4`). The demo
is a thin UI on top of the inference module rather than a separate
implementation.

## Customization tips

- **Change the model.** Update `model.py:build_model()` and adjust the Grad-CAM
  target layer in `inference.py:predict()` — `model.features[-1]` for
  EfficientNet, etc. ViT needs a different attribution method.
- **Add a "try an example" gallery.** Drop a few sample images in a
  `examples/` folder and render them as clickable buttons in the sidebar.
- **Batch inference.** Import `predict()` from `inference.py` in a separate
  script; no Streamlit required.
